      文档状态

      当前文档处于：正在更新状态

      • 待更新：有计划更新，但因为某种原因搁置了
      • 正在更新：正在每天更新中，文档内容处于巨大波动中
      • 维护：已经更新完毕，只对内容做微调和补充
      • 放弃：该技术不常用，或已过时，不再更新，未来计划删除

      目录结构

      文件结构

      • 文档项目：会有icon标注，一般同时属于导航栏里的导航或子导航。
      • 子目录：复杂的文档项目，会划分为多个子目录，简单的项目没有子目录
        • 子目录会用数字开头标注
        • 简单的项目内只有文件，没有子目录，此时的文件，也将采用数字开头标注
      • 文件：一个单独的Markdown文件
        • 文件开头会有 1-1 标注，第一个数字是目录序号，第二个是本身的序号
      ├─ Python基础  # 独立的文档项目
│  ├─ 1.入门  # 文档项目的子目录
│  │  ├─ 1-1. python介绍  # 单独的Markdown文件

      文件内的标题结构

      • 一级标题：以 罗马数字 开头
        • 当文档内容多且复杂时才会有一级标题，该标题会在侧边栏的文档名下显示
      • 二级标题：以 数字 开头
        • 简单的文档直接以二级标题开头是可以的
      • 三级标题：以 1.1 的格式开头
        • 作为二级标题的子标题，用于代替（1）
      数据类型 # 文档名
├─ Ⅰ. 数据类型介绍  # 文档的一级Title
│  ├─ 1. 简单数据类型 # 文档的二级title
│  │  ├─ 1.1. Number # 文档的三级title

      PS：一级标题前是罗马数字：Ⅰ Ⅱ Ⅲ Ⅳ Ⅴ Ⅵ Ⅶ

      PS：文档很简单的话，可以省略一级标题，直接写二级标题
